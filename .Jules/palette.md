## 2026-05-16 - Dynamic Aria Attributes on Svelte Rendered UI
**Learning:** Adding dynamic attributes like `aria-label` or conditional `aria-live` regions works perfectly in Svelte components, but verifying them with Playwright within a mocked production build requires being extremely precise with DOM element resolution. Svelte's compilation combined with custom navigation can cause standard text selectors to fail due to partial matches or hidden elements, requiring CSS-based locators or explicit evaluate scripts to reliably assert accessibility features.
**Action:** When writing Playwright a11y tests, prefer standard CSS selectors like `input[placeholder='...']` to extract `aria-` attributes, and use exact `page.locator("text=...")` or role-based matchers combined with delays to handle Svelte component lifecycle and animations accurately.
